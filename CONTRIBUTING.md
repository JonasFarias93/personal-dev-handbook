# Contributing

Este repositório é um handbook pessoal, mas sugestões são bem-vindas.

## Como contribuir
- Abra uma Issue com sugestão, correção ou melhoria
- Se possível, inclua:
  - qual arquivo (`python/testing.md`, `docker/compose.md`, etc.)
  - o que está confuso ou incorreto
  - uma proposta objetiva de ajuste

## Escopo
Este repositório prioriza:
- clareza e praticidade
- decisões explícitas
- exemplos pequenos e reutilizáveis

## O que pode ser recusado
- mudanças que aumentem burocracia
- teoria sem aplicação prática
- conteúdo que não tenha relação com o objetivo do repo

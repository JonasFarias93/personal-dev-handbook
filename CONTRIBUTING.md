# Contributing

Este repositório é um handbook pessoal de engenharia.

Sugestões são bem-vindas — desde que mantenham o foco em clareza, decisão consciente e aplicabilidade prática.

---

# 🎯 Objetivo das contribuições

Contribuições devem:

* Melhorar clareza
* Corrigir inconsistências
* Atualizar decisões já evoluídas
* Tornar padrões mais explícitos
* Reduzir ambiguidade

Este repositório **não é wiki aberta**.
Ele é um sistema de decisões pessoais documentadas.

---

# 🛠 Como contribuir

1. Abra uma Issue
2. Descreva claramente:

   * Qual arquivo está sendo discutido (`python/testing.md`, `docker/compose.md`, etc.)
   * O que está confuso, inconsistente ou incorreto
   * Proposta objetiva de melhoria
   * (Opcional) Justificativa técnica

Pull Requests são bem-vindos após alinhamento via Issue.

---

# 🧭 Critérios de aceitação

Uma contribuição tende a ser aceita quando:

* É prática e aplicável
* Mantém simplicidade
* Está alinhada com os princípios do README
* Não adiciona complexidade desnecessária
* Não contradiz decisões registradas em ADR sem justificativa

---

# 🚫 O que pode ser recusado

Podem ser recusadas contribuições que:

* Aumentem burocracia sem benefício claro
* Introduzam teoria excessiva sem aplicação prática
* Transformem o handbook em tutorial genérico
* Contrariem decisões explícitas sem argumento sólido
* Adicionem ferramentas por preferência pessoal

---

# 🗂 Governança

* Decisões estruturais devem gerar atualização em `architecture/decisions.md`
* Mudanças relevantes devem manter consistência com o restante do handbook
* Se houver conflito entre documento e prática consolidada, a prática deve ser revista e documentada

---

# 📌 Nota final

Este repositório prioriza:

* Clareza > completude
* Processo > ferramenta
* Decisão explícita > convenção implícita

Se uma contribuição melhora esses pontos,
ela é bem-vinda.
